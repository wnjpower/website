# 우앤주전력 웹사이트 전면 개편 기획 보고서

**작성일**: 2026-07-20
**대상**: www.wnjpower.com (Next.js 14 원페이지, Vercel 호스팅)
**목적**: 검색 유입 부재 + "전문 전기공사업체" 인상 부족 문제를 해소하는 전면 개편 방향 수립

---

## 1. 요약 (Executive Summary)

현재 사이트가 리드를 만들지 못하는 원인은 두 가지가 겹쳐 있다.

**① 검색에서 발견될 구조가 없다.** 인덱싱 가능한 페이지가 2개뿐인 원페이지 구조이고, 네이버 스마트플레이스·네이버 블로그·Google Search Console 어디에도 등록되어 있지 않다. 한국 로컬 검색("대구 공장 전기공사")에서 상단을 차지하는 채널(플레이스·블로그)에 존재하지 않으므로, 웹사이트를 아무리 다듬어도 네이버 유입은 구조적으로 발생하지 않는다.

**② 발견돼도 신뢰를 증명할 증거가 없다.** "전문적인 느낌이 안 남"의 실체는 디자인 스타일 이전에 **증거의 부재**다 — 시공사진 0장, 면허번호 본문 미노출, 가격표 전 항목 "협의 후 견적", 카카오톡 버튼은 더미 URL. 여기에 밝은 SaaS 랜딩페이지풍 시각 언어(파스텔 오버레이, 강조색 4종 혼용, 라운드 카드)가 산업 전기공사의 무게감과 어긋난다.

여기에 더해 **지금 문의가 들어와도 유실된다** — 계정 이관으로 Supabase/Resend가 비활성이라 폼 제출이 아무 데도 저장·발송되지 않으며, 전화·카톡 클릭은 애초에 측정되지 않는다.

**개편 전략 3축**:

| 축 | 핵심 방향 |
|---|---|
| 유입 | **삼각 편대**: 네이버 스마트플레이스(로컬 쿼리) + 네이버 블로그(정보성 쿼리) + 웹사이트 멀티페이지 전환(구글 + 전 채널의 신뢰 검증·전환 허브) |
| 전문성 | **다크 인더스트리얼 톤 재포장 + 증거(면허·실적·사진·수치) 전면 배치** — 증거 없는 리디자인은 문제를 해소하지 못함 |
| 전환·측정 | **전화 1순위 전환 경로 재설계** + 폼 인프라 복구 + 전화/카톡/폼 이벤트 측정 체계 구축 |

**실행은 4단계**: Phase 0(1주 — 유실·측정·공유 결함 즉시 수정) → Phase 1(1~2주 — 로컬 채널 등록 + 디자인 톤 정리) → Phase 2(3~6주 — 홈 재구성 + 멀티페이지 전환) → Phase 3(상시 — 시공사례·블로그 콘텐츠 축적). 최대 병목은 개발이 아니라 **사장님 협조 자료(사진·수치·계정)**이므로, §7.2의 협조 패키지를 개편 착수와 동시에 요청해야 한다.

## 2. 현황 진단

현재 사이트가 리드를 만들지 못하는 원인은 단일 문제가 아니라 **4개 축의 복합 문제**다.

### 2.1 유입 축 — 검색에 노출될 구조 자체가 없음

| 항목 | 현재 상태 | 심각도 |
|---|---|---|
| 인덱싱 가능 페이지 | **2개뿐** (`/`, `/privacy`) — 원페이지 구조라 키워드별 랜딩 페이지가 없음 | 🔴 심각 |
| 네이버 | 서치어드바이저 소유확인만 완료. **사이트맵 미제출, 스마트플레이스 미등록, 블로그 없음** | 🔴 심각 |
| 구글 | **Search Console 미등록**, Google 비즈니스 프로필 없음 | 🔴 심각 |
| OG 이미지 | `opengraph-image.tsx` 미구현, 정적 파일도 없음 → **카카오톡 공유 미리보기 깨짐** | 🔴 심각 |
| 구조화 데이터 | ElectricalContractor JSON-LD는 있으나 `image`가 존재하지 않는 `/og-image.png` 참조, `geo` 좌표·`sameAs` 공란, 면허 정보 미포함 | 🟡 보통 |
| GA4 | 코드 폴백 ID가 `'7XRHSS9V0F'`로 `G-` 접두어 누락 (`components/GoogleAnalytics.tsx`) — 환경변수 미주입 시 조용히 무효 | 🟡 확인 필요 |
| 렌더링 구조 | `app/page.tsx` 전체가 `'use client'` — 페이지별 메타데이터 분리가 불가능한 구조 | 🟡 개편 시 해소 |

**핵심**: "대구 공장 전기공사"를 검색하는 잠재 고객이 이 사이트를 만날 경로가 사실상 지명(우앤주전력) 검색뿐이다. 네이버 통합검색에서 로컬 쿼리 상단은 파워링크 → **플레이스** → **블로그** 순인데, 세 채널 중 어디에도 존재하지 않는다.

### 2.2 전문성 축 — 신뢰 요소가 전부 placeholder

사장님 피드백("전문적인 느낌이 안 남")의 실체는 디자인 스타일 이전에 **증거의 부재**다. B2B 발주 담당자가 업체를 검증할 때 보는 요소가 모두 비어 있다:

| 신뢰 요소 | 현재 상태 |
|---|---|
| 시공 현장 사진 | **0장** (`public/images/portfolio/` 빈 폴더 — 8건 실적이 텍스트 리스트로만 표시) |
| 대표 사진 | placeholder 아이콘 (`About.tsx`) |
| 전기공사업 면허번호 | Credentials 섹션에 **미노출**(`number: null`) — Footer에만 `대구-01425` (섹션 간 불일치) |
| 비용 안내 | Pricing 표 전 항목 "협의 후 견적" (실 금액 없음 — 기대를 만들고 배신하는 구조) |
| 카카오톡 채널 | 더미 URL `https://pf.kakao.com/` (FloatingCta·Contact·Portfolio 3곳) — **클릭 시 무관한 페이지로 이동** |
| 지도 | placeholder 카드 (검색 링크만) |
| 기타 | 영업시간 Header↔Contact 불일치, 팩스 미표기, 저작권 `© 2025`(현재 2026 — "방치된 사이트" 신호) |

실제 현장 이미지는 배경용 2장(factory-electrical.jpg, switchgear.jpg)뿐이다.

### 2.3 전환·측정 축 — 성과를 알 수 없는 상태

- **전화·카카오톡 클릭이 전혀 추적되지 않음.** GA4 이벤트는 폼 제출 성공 시 `generate_lead` 1개뿐. 전기공사 업종의 주 전환 경로는 전화인데, 전화 클릭 수를 모른다.
- Vercel Analytics 미설치 (`@vercel/analytics` 패키지 없음).
- **폼 제출이 실제로 아무 데도 저장·발송되지 않음**: Supabase/Resend/알림톡이 계정 이관으로 전부 비활성 (graceful degradation으로 200 OK만 반환). 지금 견적문의가 들어와도 유실된다.

### 2.4 운영 축 — 오픈 전 차단 요소 (TODO.md 6차 기준)

- Resend API 키·Supabase service_role 키 Vercel 미입력, 실제 폼 제출 미테스트
- 카카오 알림톡 외부 설정 4단계 미완 (코드는 `lib/kakao-alimtalk.ts`에 준비됨)
- `app/api/quote/route.ts`의 insert 컬럼(`company_name`, `email`, `customer_type`)이 기존 quotes 스키마에 없음 → 테이블 보강 필요
- 사장님 수령 대기 자료: 시공사진, 대표 사진, 카톡 채널 URL, 네이버 플레이스 URL, 면허 스캔, 영업시간, FAQ 검수

### 2.5 진단 결론

> 사이트는 이미 "공장·산업 전기공사 전문 B2B"로 방향을 잘 잡았고 폼·구조 등 뼈대는 갖췄다.
> 그러나 ① **검색에서 발견될 구조가 없고** ② **발견돼도 신뢰를 증명할 콘텐츠가 없으며** ③ **문의가 와도 유실되고** ④ **무엇이 효과 있는지 측정할 수 없다.**
> 전면 개편은 이 4개 축을 동시에 해소해야 한다.

## 3. 개편 전략 개요 — "삼각 편대 + 증거 기반 리브랜딩"

한국 검색 시장은 네이버·구글 이원 구조다 (국내 측정 기준 네이버 ~63% vs 구글 ~30%, 글로벌 측정 기준 팽팽). 네이버 통합검색에서 자사 웹사이트 영역은 하단 배치이므로, **웹사이트 SEO만으로는 네이버 상단 노출이 구조적으로 불가능**하다. 따라서:

```
[네이버 스마트플레이스]  ← 네이버 로컬 쿼리 상단 (지도·전화·길찾기)
[네이버 블로그]          ← 네이버 정보성 쿼리 (시공기·비용·절차)
[웹사이트 (멀티페이지)]   ← 구글 전 쿼리 + 전 채널의 신뢰 검증·전환 허브
        └─ 세 채널이 상호 링크(sameAs·프로필)로 신뢰 신호를 보강
```

- **웹사이트의 역할 재정의**: "네이버 상단 노출 수단"이 아니라, B2B 담당자가 업체 검증 단계에서 반드시 방문하는 **신뢰 검증·전환 허브**. 구글 유입(업무시간 PC 검색 B2B 담당자 비중 높음, 경쟁사 대부분 네이버만 집중 → 상대적 블루오션)은 웹사이트가 전담.
- **전문성 문제의 해법**: 디자인 스타일 변경 + **증거(사진·면허·실적·수치) 전면 배치**의 결합. 증거 없는 리디자인은 문제를 해소하지 못한다.
- **B2B 발주 담당자의 행태**: (1) "지역+공종"으로 후보 탐색 → (2) 면허·법인·실적으로 신뢰 검증 → (3) 2~3곳 견적 비교. 검증 단계에서 웹사이트를 반드시 방문하므로, 웹사이트는 "품의서에 옮겨 적을 수 있는 팩트"를 제공해야 한다.

## 4. 유입 전략 (검색 노출)

### 4.1 원페이지 → 멀티페이지 IA 전환

**설계 원칙**
1. **1 키워드 클러스터 = 1 페이지.** 현재는 10여 개 키워드가 홈 1페이지에 몰려 관련성 점수가 분산된다.
2. **기존 데이터 구조 재사용.** `content/services.ts`의 4분류(`factory`/`power`/`panel`/`interior`)와 `content/faq.ts`의 `segments` 필드가 이미 페이지 분리를 전제한 구조 — URL 슬러그로 그대로 승격.
3. **슬러그는 영문 소문자.** 키워드 타겟팅은 URL이 아닌 title·H1·본문이 담당.
4. **렌더링 구조 전환.** `app/page.tsx`의 `'use client'` 전체 선언을 해체하고 견적폼만 클라이언트 컴포넌트로 남겨, 각 페이지가 서버 컴포넌트 + `generateMetadata`를 갖도록 한다.

**페이지 목록·타겟 키워드**

| URL | 페이지 | 주 타겟 키워드 | 콘텐츠 소스 |
|---|---|---|---|
| `/` | 홈 | 대구 전기공사, 대구 전기공사 업체, 우앤주전력 | 현행 원페이지 축약 (Hero+요약+CTA) |
| `/services/factory` | 공장·산업 전기공사 | 대구 공장 전기공사, 경북 공장 전기, 공장 신축 전기공사, 동력설비 공사 | services.ts factory + FAQ industrial |
| `/services/power` | 수전설비·계약전력 증설 | 계약전력 증설, 수전설비 공사, 전기 증설, 승압 공사 | services.ts power + FAQ industrial |
| `/services/panel` | 배전반·분전반 제작 | 배전반 제작, 분전반 제작, 배전반 제작 업체, 분전함 교체 | services.ts panel |
| `/services/interior` | 인테리어·일반 전기 | 대구 인테리어 전기, 상가 전기공사, 카페 전기공사 | services.ts interior + FAQ interior |
| `/portfolio` | 시공사례 목록 | 대구 전기공사 시공사례 | portfolio.ts |
| `/portfolio/[slug]` | 시공사례 상세 (8건~) | 롱테일 (예: 칠곡 공장 전기공사) | portfolio.ts 확장 + 사진 |
| `/about` | 회사소개·면허 | 우앤주전력, 대구 전기공사업 면허 업체 | About+Credentials 이관 |
| `/faq` | 자주 묻는 질문 | 질문형 키워드 + FAQPage 스키마 | faq.ts |

- 견적폼은 별도 페이지가 아니라 **모든 서비스·사례 페이지 하단에 임베드** — B2B 리드는 콘텐츠 소비 직후 전환율이 가장 높다.
- **지역 페이지(`/areas/구미` 등)는 1단계에서 제외.** 고유 콘텐츠 없이 지역명만 치환하면 구글 도어웨이 페이지 판정 위험. 지역별 사례 3건 이상 축적 후 도입.
- 내부 링크: 홈→서비스 4페이지, 서비스→해당 카테고리 사례, 사례→서비스(문맥 링크), FAQ→서비스. 하위 페이지 전체 BreadcrumbList.

### 4.2 키워드 매핑 체계

검색량 실측은 **네이버 검색광고 키워드도구**(광고 집행 없이 무료 조회 가능)와 네이버 DataLab으로 구현 단계에서 검증. 가설 매핑:

| 계층 | 성격 | 키워드 예시 | 담당 페이지 |
|---|---|---|---|
| 헤드 (검색량 有, 경쟁 高) | 지역+업종 | 대구 전기공사, 배전반 제작 | 홈, /services/panel |
| 미드 (상업 의도 高) | 지역+공종 세분 | 대구 공장 전기공사, 계약전력 증설, 수전설비 공사, 구미 공장 전기공사 | 서비스 4페이지 |
| 롱테일 (전환율 最高) | 질문·상황형 | 공장 계약전력 증설 비용, 3상 전기 증설, 한전 불입금, 노후 배전반 교체 | 사례 상세, FAQ, 블로그 |

- 기존 사례 8건의 위치(칠곡×2, 달성, 경산, 달서구×2, 중구, 수성구)가 지역×공종 매트릭스의 씨앗. 향후 칠곡·구미·경산·성주·달성(공단 밀집) × 공장 신축/증설/수전/배전반 조합을 우선 채운다.
- **B2B 헤드 키워드는 절대량이 작아도 건당 계약이 수천만 원대이므로 월 수십 회 검색 롱테일도 채산성이 성립** — 우선순위 판단 기준.
- 분기마다 GSC·서치어드바이저 실제 유입 쿼리 확인 → 노출 有·순위 低 쿼리를 다음 분기 콘텐츠로 승격.

### 4.3 네이버 전략 (제1 채널)

**① 스마트플레이스 — 최우선.** 로컬 쿼리에서 광고 다음 최상단이 플레이스 팩. 미등록은 네이버 유입 기회의 최대 지분을 포기하는 것.
- 사장님에게 기존 플레이스 존재 여부 확인(사업자등록 시 자동 생성된 미클레임 플레이스 가능성) → 없으면 신규 등록. 신규 등록 후 90일 '신규' 라벨 부스팅 골든타임에 정보·사진·리뷰 집중.
- NAP(상호·주소·전화)를 플레이스=웹사이트 푸터=JSON-LD=카카오맵=구글에 완전 일치.
- 상세설명에 타겟 키워드 자연 포함("대구·경북 공장 전기공사 / 수전설비·계약전력 증설 / 배전반·분전반 자체 제작 / 전기공사업 면허 대구-01425"), 시공·배전반 제작 사진 10장+ (가로 1200px+).
- 2025년 개편 후 플레이스 랭킹은 리뷰 수보다 **실사용자 행동(전화·길찾기·저장·체류)**이 핵심 신호 — 모든 접점(명함·세금계산서 메일·블로그)에 플레이스 링크 배치. 리뷰 대행·조작 금지(적발 시 노출 급락).
- 확보된 플레이스 URL을 `SchemaOrg.tsx` `sameAs`와 푸터에 반영.

**② 서치어드바이저 사이트맵 제출** (10분): `sitemap.xml` 제출 + 멀티페이지 오픈 시 개별 URL 수집 요청. 색인 반영 통상 2주. 직접 유입보다 지명 검색 대응·신뢰 기반 확보 목적.

**③ 네이버 블로그 — 정보성 쿼리 담당.** "계약전력 증설 비용" 류 쿼리 상단은 블로그 영역이며 웹사이트가 대신할 수 없다.
- C-Rank·D.I.A. 알고리즘 대응: 주제 '전기공사' 일관 유지, 직접 시공 현장 사진·수치·과정 중심 경험 글, 초기 3개월 주 1회 발행.
- **이중 발행**: 시공사례 1건을 (a) 웹사이트 `/portfolio/[slug]` — 구글용 표준 템플릿 (b) 네이버 블로그 — 스토리텔링 톤, **서로 다른 원고**로. 동일 원고 복붙은 중복 문서 페널티 위험. 블로그 말미에 견적폼 링크.

**④ 보조**: 지식iN 전문가 답변(낮은 공수, 우선순위 낮음). 콘텐츠 축적 3개월 공백을 메우는 브리지로 파워링크·플레이스광고 소액 테스트 검토 — 유료, 선택 (§8).

## 5. 전문성·디자인 개편 전략

### 5.1 왜 전문적으로 안 보이는가 — 코드 레벨 진단 5가지

1. **히어로에 "현장"이 없다.** 유일한 현장 사진(factory-electrical.jpg) 위에 좌측이 거의 불투명한 **흰색 그라데이션**(`.bg-photo-hero`)을 덮어, 결과물은 "연회색 배경 + 파란 글씨"의 전형적인 SaaS 랜딩 히어로다. 잘 만든 시공업체 사이트는 정반대 — 어두운 대형 현장 사진이 화면을 지배하고 텍스트가 그 위에 얹힌다.
2. **강조색이 4종 혼용된다.** `#0A3D91`(본문 블루) + `#FF5500`(전화 CTA) + `#FFB800`(공지띠·다크 섹션 액센트) + `#C8581F`(인테리어 카드 테라코타). 스크롤마다 다른 강조색이 나타나면 "브랜드가 통제되고 있다"는 인상이 사라진다. 전문성은 곧 일관성이다.
3. **디자인 토큰과 실제 색이 어긋난다.** `globals.css`의 `--primary: oklch(0.49 0.27 264)`는 주석("WNJ Blue #0A3D91")과 달리 실제 #0A3D91(≈ oklch(0.39 0.15 260))보다 훨씬 밝고 채도 높은 블루 — shadcn 기본 컴포넌트(체크박스·포커스 링)와 하드코딩된 `#0A3D91` 요소의 색이 미묘하게 다르다. 사용자는 이유를 몰라도 "어딘가 어설프다"고 감지한다.
4. **신뢰의 시각적 밀도가 낮다.** 히어로 신뢰 스탯 4카드("전기공사업 면허 / 업력 20년+ / 배전반 자체제작 / 수전·동력설비")가 전부 **번호·수치 없는 형용사형 주장**. 면허번호는 Footer에만 있다.
5. **라운드·파스텔·애니메이션 과잉.** `rounded-2xl/3xl` 카드, 연한 틴트 배경, 호버 부양, 블루프린트 그리드 드리프트 — "귀엽고 친절한" 문법. 산업 B2B의 신뢰 언어는 **직선적 구조, 굵은 괘선, 수치 중심 타이포**다.

### 5.2 디자인 방향 — "인더스트리얼 코퍼레이트" (추천)

업계 레퍼런스 공통 패턴 (리서치): 딥 네이비/차콜 베이스 + 안전 오렌지 계열 **단일 강조색**, 다크 테마는 상업·산업 시공사의 최근 주류 트렌드, **자체 촬영 현장 사진이 디자인의 중심축**(전문 사진 보유 사이트가 유의미하게 높은 클릭 확보), 굵은 헤드라인 + 수치 강조 타이포, 첫 화면에서 "누구인가/무엇을/어디서/어떻게 연락" 즉답.

| 옵션 | 설명 | 판정 |
|---|---|---|
| A. 클린 코퍼레이트 (현행 강화) | 밝은 배경 유지, 색만 정리 | 공수 최소지만 "가벼움"이라는 근본 원인이 남음 |
| B. 풀 다크 인더스트리얼 | 전 섹션 차콜/블랙 | 사진 0장 상태에선 어두운 빈 화면. 폼·표 가독성 저하 |
| **C. 인더스트리얼 코퍼레이트 ✅** | **다크 히어로 + 다크 신뢰/견적 섹션, 본문은 라이트 유지하되 직선·수치 중심 재조정** | 무게감 확보 + 가독성 유지 + 기존 컴포넌트 재활용. 현재도 Credentials·QuoteSection이 다크(#0B1220)이므로 "다크 = 신뢰·전환 구간" 리듬을 히어로까지 확장하면 구조 변경 없이 톤 전환 가능 |

**구체 스펙**:
- **색**: Base Dark `#0B1220` 유지 / Primary `#0A3D91` 유지하되 `globals.css` 토큰(oklch)을 실제 값으로 교정 / **Accent를 안전 오렌지 `#FF5500` 1종으로 단일화** — `#FFB800`·`#C8581F` 퇴출 (카카오 `#FEE500`만 브랜드색 예외). 안전 오렌지는 안전조끼·안전모와 연결돼 "현장" 연상이 가장 강함.
- **타이포**: Pretendard 유지. 헤드라인 800 웨이트 + tracking-tight, **수치는 본문 대비 2.5~3배 + tabular-nums**, 면허·사업자번호는 현행 `font-mono` 유지(잘한 부분).
- **형태**: 카드 라운드 `rounded-2xl/3xl` → `rounded-md/lg` 축소, 호버 부양·블루프린트 드리프트 애니메이션 제거. 모션은 CountUp(실적 수치)에만 집중.
- **사진 톤**: 저채도·고대비·약간 차가운 화이트밸런스로 보정 프리셋 통일. 오버레이는 흰색 워시 폐기, `rgba(11,18,32,0.55~0.75)` 네이비-차콜 계열로 일원화.

### 5.3 신뢰 아키텍처

**B2B 발주 담당자의 평가 기준** (시설관리자용 컨트랙터 선정 가이드 공통): ① 면허·등록의 **검증 가능성** ② 동종·동규모 실적(용량 kW·면적·공기) ③ 항목별 견적 투명성 ④ 안전관리 체계·보험 ⑤ 커뮤니케이션·공정관리 ⑥ 준공 후 A/S. 한국에서 ①의 실행 수단은 **전기공사협회 전기공사종합정보시스템(ECIC) 조회** — 면허번호 노출은 장식이 아니라 "검증 프로토콜의 시작점"이다.

**사진 확보 전/후 단계별 대응**:
- **사진 없이 즉시 (1주 내)**: 면허번호 전면 노출 + 조회 딥링크 교정 + 포트폴리오를 "시공 실적표"로 전환(B2B 담당자는 갤러리보다 표를 더 신뢰) + 면허증·사업자등록증 스캔 PDF 열람(스캔은 촬영보다 쉬움) + 미확인 수치("업력 20년+" 등) 전건 사장님 컨펌.

### 5.4 콘텐츠·카피 방향

**진단**: 현행 카피는 깔끔하지만 **"주장형"**("책임집니다", "믿고 맡길 수 있는 이유"). B2B 담당자는 주장이 아니라 **품의서에 옮겨 적을 수 있는 팩트**를 찾는다. Pricing 섹션은 "비용이 궁금하신가요?"라고 묻고 전 항목 "협의 후 견적"으로 답하는 — 기대를 만들고 배신하는 구조.

**B2B 카피 원칙**: ① 형용사 1개 지울 때마다 수치 1개 투입 ② 독자를 "발주 담당자"로 상정 — 그가 상사에게 보고할 문장을 대신 써줌 ③ 프로세스·문서(견적서 양식·준공 서류·A/S 조건)를 구체적으로 언급 ④ "대구·경북" 반복보다 "성서·달성·왜관 산업단지" 같은 실제 권역명.

**섹션별 개선 예시** (N은 사장님 확인 후 치환):

| 위치 | 현행 | 개선안 |
|---|---|---|
| Hero 헤드라인 | "대구·경북 공장 전기공사, 우앤주전력이 책임집니다" | **"공장 전기공사, 면허·제작·시공을 한 회사가 끝냅니다"** + 서브: "전기공사업 등록 대구-01425 · 수전설비·동력·배전반 자체 제작 — 현장 견적 무료" |
| Hero 스탯 | 형용사 4개 | "누적 시공 N건 · 배전반 제작 N면 · 계약전력 증설 N건 · A/S 당일 출동" |
| Credentials 제목 | "믿고 맡길 수 있는 이유" | **"서류로 확인하세요"** — 조회 가능성 자체를 헤드라인으로 |
| Pricing | "협의 후 견적" 표 | **"견적은 이렇게 산출합니다"**: 산출 항목(자재비/노무비/경비) 공개 + 익명화된 실제 견적 예시 1~2건("○○공단 식품공장 증설, 계약전력 100→300kW, 공기 12일") + "견적서는 항목별 내역서로 드립니다" |
| About | "제 집 공사라는 마음으로" | 유지(정서 담당)하되 아래에 팩트 블록: 보험·상주 인력·보유 장비 |
| 폼 위 | "1영업일 내 연락드립니다" | **"영업시간 내 접수 시 당일, 그 외 다음 영업일 오전 연락"** — 응답 속도가 곧 역량 신호 |

### 5.5 전환 경로 최적화

**채널 우선순위를 전화 1순위로 재편.** 로컬 서비스·시공 업종에서 전화 리드는 폼 대비 성약률이 수 배~십수 배 높고, 사양이 복잡한 B2B 공사일수록 전화 선호가 강하다. 현재 사이트는 시각 위계상 폼이 1순위 — 재편안:
- Hero 1차 CTA를 **전화 버튼(오렌지, 번호 노출)**로, 2차를 "견적문의 남기기"로.
- FloatingCta: 현행 우하단 FAB 2개(600px 스크롤 후 표시) → **모바일 하단 고정 바(전화 50% + 견적 50%), 즉시 표시**. 시공업체 모바일 표준 패턴, 탭 면적 대폭 확대.
- 카톡 버튼은 채널 개설 전까지 숨김.

**4-STEP 폼 개선** (현행 평가: 필수 3필드 최소화·전화 하이픈 자동화·honeypot 등 기본기 우수. 단 화면상 입력 요소 9개로 B2B 최적 상한선):
1. **파일 첨부 추가 (도면·현장 사진·기존 분전반 사진)** — 견적 정확도와 리드 품질을 동시에 올리는 단 하나의 필드. "사진 한 장이면 견적이 빨라집니다" 카피 병기. Supabase Storage 무료 tier 활용.
2. 이메일·시공 지역 필드 제거 또는 접기 — 상세내용에 흡수.
3. **"전화 콜백" 퀵 패스 신설**: 폼 상단에 "복잡하면 번호만 남기세요 — 저희가 겁니다" (연락처 1필드 + 희망 시간대). 폼 이탈층 회수 장치.
4. 제출 버튼 아래 응답 SLA 명시 + 접수 완료 화면에 담당자명·직통번호.

**CTA 배치**: Hero CTA가 11개 섹션 아래(#quote)로 점프하는 현행 구조 보완 — 섹션 4~5개마다 **컨텍스트 CTA 밴드**(예: Portfolio 뒤 "비슷한 공사를 계획 중이신가요? — 전화 / 견적") 삽입. FAQ 하단에 전화 CTA 재배치.

**마이크로 전환 (견적 전 단계의 낮은 허들)**:

| 마이크로 전환 | 설명 |
|---|---|
| **회사소개서 PDF 다운로드** | 발주 담당자가 내부 품의에 첨부할 2~4p 자료(면허·실적표·보험·연락처). B2B에서 가장 강력 — 다운로드한 사람은 검토 단계 리드 |
| 면허·등록증 사본 열람 | 게이트 없이 공개 |
| 공장 전기공사 준비 체크리스트 | "증설 전 확인할 7가지" 1p — SEO 겸용 |
| 전화 콜백 신청 | 위 퀵 패스 |
| 카톡 채널 추가 | 채널 실개설 후, "느슨한 연결" 채널로 포지셔닝 |

다운로드 이메일 게이트는 초기에는 걸지 않고 **게이트 없이 공개 + GA4 다운로드 이벤트만 추적** 권장.

## 6. 측정 체계

개편 효과를 판정하려면 **개편 전 베이스라인부터** 측정이 시작돼야 한다. 즉시 구축:

| 이벤트 | 트리거 | 구현 위치 |
|---|---|---|
| `phone_click` | 모든 `tel:` 링크 클릭 (위치 라벨: hero/header/floating/contact/…) | `gtagEvent` 재사용 — Header·Hero·Contact·Footer·QuoteSection·FloatingCta·QuoteForm의 tel 링크 전체 |
| `kakao_click` | 카톡 채널 링크 클릭 | 채널 개설 후 |
| `generate_lead` | 폼 제출 성공 (기존) | `QuoteForm.tsx` — 유지 |
| `callback_request` | 콜백 퀵폼 제출 | 신설 시 |
| `pdf_download` | 회사소개서·면허증 PDF 열람 | 신설 시 |

- **GA4 ID 결함 수정** (P0-1): 폴백 `'7XRHSS9V0F'` → `'G-7XRHSS9V0F'`, GA4 실시간 보고서로 실측정 여부 검증.
- **Vercel Analytics 활성화**: `@vercel/analytics` 설치 + `<Analytics />` — 무료 tier, GA4와 상호 검증용.
- **폼 인프라 복구가 측정보다 선행**: Supabase 신규 프로젝트 키 입력 + quotes 테이블 보강(`company_name`·`email`·`customer_type` 컬럼) + Resend 키 입력 + 실제 제출 E2E 테스트. **지금은 리드가 물리적으로 유실되는 상태.**
- KPI 정의: 주간 전화 클릭 수 / 폼 제출 수 / (개설 후) 카톡 유입 수 / GSC·서치어드바이저 노출·클릭 / 플레이스 전화·길찾기 수. 월 1회 리포트 루틴.

### 7.2 사장님 협조 패키지 (개편 착수와 동시 요청 — 최대 병목)

**"1회 현장 동행 촬영 반나절 + 서류 스캔 + 수치 컨펌"으로 묶어 한 번에 요청 권장.**

**A. 촬영 (우선순위순)**
1. 최우선 5~8컷: **배전반 앞 작업 장면**(안전모·조끼 착용, 계측기 사용 — 히어로 배경 제1후보), 공장 내부 전경+전기설비 와이드, 대표 프로필(현장 반신), 팀 단체(작업복 통일)
2. 포트폴리오용 (대표 실적 3~5건 × 2~3컷): **배전반 제작 3단계**(외함→결선→완성품 — "자체 제작" 주장의 유일한 증거), 수전설비, 동력 배선 전/후, **노후 분전반 교체 전/후 같은 앵글**(전환력 최상), 인테리어 조명 완성컷
3. 촬영 팁: 스마트폰 가로, 보정은 개발 측 일괄. **전 작업컷 안전장구 착용 필수**(안전관리 능력의 시각 증거). 고객사 상호·얼굴 식별 컷은 동의 확인, 상호는 "○○공단 자동차부품 공장" 익명화 원칙. 각 사진에 메모 1줄(장소·공종·규모·연월) — 이것이 portfolio.ts 데이터가 됨.

**B. 서류·수치**
